# Name Frequency Counter

Консольная утилита на Go, подсчитывающая частоту имён в файле.  
Принимает текстовый файл (по одному имени на строку), выводит отсортированный список имён с частотой.

## Возможности

- Подсчёт количества вхождений каждого имени
- Сортировка по алфавиту или по частоте
- Направление сортировки (`asc`, `desc`)
- Нормализация регистра (по умолчанию: `алЁНа` → `Алёна`)
- Сохранение исходного регистра (`--preserve-case`)
- Вывод результата в файл (`-o`)
- Потоковая обработка больших файлов

## Установка и сборка

go build -o namecount ./cmd/namecount   # здесь и далее namecount.exe для Windows

## Использование

```bash
./namecount [опции] <имя_входного_файла>
```

## Опции

| Флаг             | Значение по умолчанию          | Описание                                         |
|------------------|-------------------------------|--------------------------------------------------|
| `--sort-by`      | `freq`                        | Тип сортировки: `alph` (по алфавиту) или `freq` (по частоте) |
| `--order`        | `asc` для `alph`, `desc` для `freq` | Направление сортировки: `asc` или `desc`    |
| `--preserve-case`| `false`                       | Сохранять исходный регистр имён (не нормализовать) |
| `-o`             | вывод в stdout                | Путь к выходному файлу                           |

## Примеры

Входной файл `names.txt`:

```
Алёна
Миша
Алёна
Дима
```

### 1. По умолчанию (по частоте, по убыванию)

```bash
./namecount names.txt
```

```
Алёна:2
Дима:1
Миша:1
```

### 2. По алфавиту, по возрастанию

```bash
./namecount --sort-by alph names.txt
```

```
Алёна:2
Дима:1
Миша:1
```

### 4. По алфавиту, по убыванию

```bash
./namecount --sort-by freq --order desc names.txt
```

```
Миша:1
Дима:1
Алёна:2
```

### 4. По частоте, по возрастанию

```bash
./namecount --sort-by freq --order asc names.txt
```

```
Дима:1
Миша:1
Алёна:2
```

### 5. Сохранить исходный регистр (отключить нормализацию)

```bash
./namecount --preserve-case names.txt
```

### 6. Вывод в файл

```bash
./namecount -o result.txt names.txt
```

## Особенности обработки регистра

По умолчанию имена приводятся к нормальной форме для русского языка (первая буква заглавная, остальные строчные):
«алёна», «Алёна», «алЁНа» -> "Алёна".
Это гарантирует, что одинаковые имена, записанные в разном регистре, считаются одним именем.
Флаг --preserve-case отключает такое приведение, тогда регистр сохраняется как в исходном файле.

## Примечания по реализации

- Используется потоковое чтение `bufio.Scanner`, что позволяет обрабатывать файлы любого размера без полной загрузки файла в память.
- Максимальная длина одной строки по умолчанию — 64 КБ (`MaxScanTokenSize` в `bufio.Scanner`), то есть от 16384 до 65536 символов в зависимости от их размера в UTF-8 (1-4 байта), чего должно быть достаточно для обычных имен.
- Для нормализации регистра применяется пакет `golang.org/x/text/cases` с локалью `language.Russian`. Это корректно обрабатывает кириллические имена и стандартный Title‑case для латиницы. При необходимости поддержки других алфавитов локаль можно параметризовать.
- При одинаковой частоте имена сортируются по алфавиту.
- Флаг `--preserve-case` позволяет отключить нормализацию и сохранить исходный регистр имён.
